Functional Requirements (FR)

1. FR1: The chatbot must allow users to book appointments.

2. FR2: The chatbot must allow users to cancel appointments using phone number + appointment ID.

3. FR3: The chatbot must allow users to reschedule appointments.

4. FR4: The chatbot must answer FAQs like clinic hours, services, and pricing.

5. FR5: The chatbot must verify identity for cancellation/rescheduling.

6. FR6: The system must store appointments with unique Appointment IDs.

7. FR7: The system must notify the user if a requested time slot is unavailable.

8. FR8: The chatbot must collect patient information (name, phone number, service type).

5. NFR5: Scalability: System should support increasing number of users.

6. NFR6: Reliability: Appointment confirmations must be accurate with no duplication.

7. NFR7: Cross-Platform: Chatbot must work on desktop and mobile.
